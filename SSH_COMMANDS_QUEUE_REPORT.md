## 🎯 Обзор

Добавлена полная поддержка очереди для SSH команд, что позволяет выполнять длинные SSH операции (которые могут работать часами) через систему очереди с отслеживанием прогресса и статуса.

## 🆕 Новые возможности

### 1. Поддержка очереди для SSH команд
- **Автоматическая очередь** - команды с timeout > 60 секунд автоматически используют очередь
- **Ручное управление** - параметр `use_queue` для явного указания использования очереди
- **Отслеживание прогресса** - мониторинг выполнения длинных SSH операций
- **Управление задачами** - возможность отмены и мониторинга SSH задач

### 2. Новые типы задач
- **SSH_CONNECT** - подключение к SSH серверу
- **SSH_EXEC** - выполнение команд через SSH

### 3. Коды ошибок SSH
- `SSH_CONNECTION_FAILED` - ошибка подключения
- `SSH_AUTHENTICATION_FAILED` - ошибка аутентификации
- `SSH_KEY_NOT_FOUND` - ключ не найден
- `SSH_PERMISSION_DENIED` - доступ запрещен
- `SSH_EXEC_FAILED` - ошибка выполнения команды
- `SSH_TIMEOUT` - превышение времени ожидания
- `SSH_HOST_UNREACHABLE` - хост недоступен

## 🎯 Использование

### Примеры команд

```bash
# Короткая команда (прямое выполнение)
curl -X POST http://localhost:20001/cmd \
  -H "Content-Type: application/json" \
  -d '{
    "command": "ssh_exec",
    "host": "localhost",
    "command": "echo Hello World",
    "timeout": 10,
    "use_queue": false
  }'

# Длинная команда (через очередь)
curl -X POST http://localhost:20001/cmd \
  -H "Content-Type: application/json" \
  -d '{
    "command": "ssh_exec",
    "host": "remote-server",
    "command": "long-running-script.sh",
    "timeout": 3600,
    "use_queue": true
  }'

# Автоматическая очередь (timeout > 60s)
curl -X POST http://localhost:20001/cmd \
  -H "Content-Type: application/json" \
  -d '{
    "command": "ssh_exec",
    "host": "remote-server",
    "command": "backup-database.sh",
    "timeout": 7200,
    "use_queue": false
  }'

# С SSH ключом
curl -X POST http://localhost:20001/cmd \
  -H "Content-Type: application/json" \
  -d '{
    "command": "ssh_exec",
    "host": "remote-server",
    "command": "deploy-application.sh",
    "username": "deploy",
    "key_path": "~/.ssh/deploy_key",
    "timeout": 1800,
    "use_queue": true
  }'
```
